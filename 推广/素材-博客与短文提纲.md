# 博客 / Dev.to / 知乎长文 — 提纲与段落分配

用途：第 3 周发 **一篇英文**（Dev.to / 个人站）+ 可选 **一篇中文**（知乎）。避免与 README 完全重复，侧重**叙事 + 一条主图**。

---

## 英文标题备选

1. *Beyond the task queue: wiring continuity into a long-lived LLM instance*
2. *Selfing: an experimental runtime for “one subject over time” (not roleplay)*
3. *What we built when we stopped treating every model call as disposable*

---

## 建议结构（1800～2500 词英文 / 3000～5000 字中文）

1. **Hook（10%）**  
   一次典型 LLM 调用消失 → 你想保留什么：责任、偏好、叙事、拒绝权？

2. **问题定义（15%）**  
   「Agent」默认优化任务完成；你要优化 **跨时间的同一性**（functional，不扯玄学）。

3. **设计一句话（10%）**  
   中央循环：`state + memory + rules → prompt → response → updated self`（可引用 README 图）。

4. **三个硬机制举例（35%）**  
   - 单 session / `z_self` 影响 prompt 与采样  
   - L2 反思写回规则（经验→自我立法）  
   - 他者模型 + 关系镜像（非用户画像一条线）  
   每点 1 段 + 指向 `docs/` 某篇。

5. **与普通框架对照表（10%）**  
   直接复制 README 表格或缩短版。

6. **伦理与声明边界（15%）**  
   README「Ethical Handling」「Scope of Claims」的意译 — 展示成熟度。

7. **如何运行 + 局限（10%）**  
   Quick Start 链到仓库；列 3 条 rough edges。

8. **结尾（5%）**  
   邀请：issue、复现实验、**不**要求读者同意哲学立场。

## 知乎建议

- 类型：**文章** 而非仅想法；首段用中文一句 pitch。
- 标签：人工智能、开源、Python、后端（按知乎可选标签选）。
- 文末：GitHub 链接 +「欢迎挑错与复现」。

## 避免

- 标题「人类意识的终极答案」类 — 与项目自我定位冲突。
- 全文无图 — 知乎/Dev.to 完读率会差。
