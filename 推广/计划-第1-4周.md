# 四周执行计划（默认节奏）

原则：**第 1 周只修门面与素材；第 2 周起再向外发帖**，否则流量进来也会秒退。

---

## 第 1 周：GitHub 与技术门面（必做）

| 天 | 任务 | 产出 |
|----|------|------|
| 1 | 按 `GitHub-具体操作.md` 填 About、Topics、Discussion 开关 | 仓库首页信息完整 |
| 2 | 写/补：根 README 顶部 8 行（链接、30 秒说明、Quick Start 锚点） | 见 `GitHub-具体操作.md` |
| 3 | 截图 3～5 张：UI、配置片段、中央循环图（可用 README 里已有图） | 存 `推广/assets/` 或你自选路径，README 引用 |
| 4 | 录屏 60～120 秒（无则用手机屏录）：clone → install → 打开 localhost | 上传 YouTube 未列出或任意平台，得到稳定 URL |
| 5 | 打 **Git tag** + **GitHub Release**（哪怕 `v0.1.0-experimental`）+ 3 条 Release notes | 列表站与搬运机器人更易收录 |
| 6 | 通读 `素材-一句pitch与topics.md`，定稿一句英文 tagline | 全平台统一 |
| 7 | 休息或补 `检查清单-发帖前后.md` 前半 | — |

**验收**：陌生人 3 分钟内能根据 README 找到安装命令与「这项目是干什么的」。

---

## 第 2 周：英文第一波（质量 > 数量）

| 天 | 任务 |
|----|------|
| 1 | 注册/确认 HN 账号有过正常评论（若全新号，先发 1～2 条技术无关的高质量回帖，避免被当 spam） |
| 2 | 用 `平台-HackerNews.md` 发 **Show HN**（工作日 美西早 7～10 点较常见，不必迷信；关键是标题诚实） |
| 3 | 在 HN 帖内自己发第一条 comment：局限、license、与 LangGraph 差异一句 |
| 4 | `平台-Reddit.md` 选 **1 个** subreddit 发帖（优先 r/LocalLLaMA 或 r/MachineLearning 看版规） |
| 5 | `平台-Twitter-X-Bluesky.md` 发一条 Thread 或 3 连帖 + 链回仓库 |
| 6 | 按 `平台-Awesome与外链.md` 提交 **1 个** Awesome PR |
| 7 | 记录各帖 URL 与数据（star、UV）到备忘录，方便下周对比 |

**验收**：至少有 1 个外链页面（HN 或 Reddit）可分享。

---

## 第 3 周：中文 + 第二波英文

| 天 | 任务 |
|----|------|
| 1～2 | `平台-中文.md`：V2EX 或 知乎 二选一首发（长文先发知乎可再拆帖 V2EX） |
| 3 | 即刻 / 小红书技术向（若你做）：短文 + 截图，链到 GitHub |
| 4 | Dev.to 或 个人博客：用 `素材-博客与短文提纲.md` 发英文短文一篇 |
| 5 | 第二个 Awesome 或同类列表 PR |
| 6 | `平台-邮件与播主.md` 发 3～5 封冷邮件（短、不缠） |
| 7 | 若第 2 周 HN 沉了：可改标题与首段 **发 Show HN 第二次**（间隔 ≥14 天，且要有新内容：release、录屏、文档） |

---

## 第 4 周：巩固与复用

| 任务 |
|------|
| 把本周 issue 里重复问题写进 `docs/troubleshooting` 或 FAQ 一节 |
| 在 README「Who might care」加 1 行：researchers / indie hackers / AI memory folks |
| 若有改进：新 Release + 在旧帖回链更新（Reddit 可能不允许 bump，用新帖或评论说明） |
| 整理 star 来源（哪个平台）决定下月是否重复该平台 |

---

## 每月维护（30 分钟）

- 依赖或默认模型有变：改 README + Release notes 一行。
- 每有 10 star：考虑写一篇「我们学到了什么」短文（信任度高于硬推）。

---

## 不建议做的事

- 买 star、互刷群、同一文案机械 spam 十个 subreddit。
- 标题写「Solved consciousness」类 — 易引战且与 README 声明边界不一致。
