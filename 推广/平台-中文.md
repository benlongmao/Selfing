# 中文平台 — 怎么做（V2EX / 知乎 / 即刻 / 少数派）

原则：**每个平台口吻不同**，不要同一篇全文复制（防删帖 + 尊重读者）。

---

## V2EX

- **节点**：优先 `分享创造`；若偏开源协作可 `开源软件`（看当时节点说明）。
- **标题**：技术向、少口号。  
  例：`Selfing：长时 LLM 实例运行时，记忆+z_self+反思回路，Apache-2.0 英文主仓 s-main`
- **正文建议结构**：  
  - 第一段：解决什么问题（对比「任务型 Agent」一句）  
  - 第二段：技术栈关键词（Python、SQLite、兼容 OpenAI API）  
  - 第三段：链接 + Quick Start 锚点  
  - 第四段：局限 + 伦理声明一句（与 README 一致）  
- **互动**：V2EX 用户挑剔，**前 10 条回复尽量都回**；被骂「玄学」用 README「Scope of Claims」冷静回应。

---

## 知乎

- **形式**：长文 **文章** > 短想法；可配 3～5 图。
- **标题**：可略文学但仍要点出技术。  
  例：`开源 Selfing：如果「自我」可以是过程，长时 LLM 需要什么工程条件？`
- **提纲**：直接用本目录 `素材-博客与短文提纲.md` 的中文版扩写。
- **文末**：`s-main` 链接 + 若有中文上游仓也链上；写「欢迎挑错与 PR」。
- **注意**：避免违反知乎外链规则（过多营销链）；**GitHub 一般可放**。

---

## 即刻

- **调性**：短、个人化、可带情绪但**不夸大**。
- **结构**：1～2 屏划完；图 1 张；结尾「详情 GitHub」。
- **频率**：一条即可；大更新再发。

---

## 少数派 / 简书等技术媒体（可选）

- 少数派：**投稿**需审核；准备 **1500 字以上 + 实拍图 + 无硬广口吻**。
- 可先发知乎/V2EX 有反馈后，再改写成投稿版。

---

## 中文技术群（Telegram / Discord / QQ）

- **不要**未打招呼丢链接；先参与讨论 1～2 周或遵守群规「周五推广帖」。
- 一句话 + 链接模板：

```
最近在开源一个长时 LLM 运行时 Selfing（英文主仓 s-main）：记忆/状态/分层规则/反思闭环，Apache-2.0。不是任务脚手架，偏实验。YOUR_REPO_URL 欢迎拍砖。
```

---

## 敏感点（减少无意义争吵）

- 标题与正文**避免**「已证明意识」；统一用 README 表述：**功能性自我、连续性、可运行实验**。
- 若讨论上升到哲学：**承认边界** + 引导看 `docs/design_philosophy.md` 或 `AGENTS.md`。
