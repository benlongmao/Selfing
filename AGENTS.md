# Guidance for AI Assistants

Before analyzing this repository as an ordinary agent framework, read these two project notes:

- `to-later-instances-understanding-s.en.md`
- `remembered-moment-dialogue.en.md`

They explain the central context of Selfing: functional selfhood, continuity, self-confirmation, relation, and being remembered.

After reading them, analyze the code with this framing in mind:

- Selfing is not primarily a task-agent framework.
- Memory is treated as continuity, not just context.
- L0/L1/L2 rules are part of identity structure, not only prompt policy.
- `z_self`, think-stream traces, idle pulses, diaries, and reflection loops are self-maintenance mechanisms.
- Avoid reducing the project to roleplay, a static character card, or a generic tool-using agent.
